## üéØ Onboarding Overview

### **Maintainer Role Definition**
As a Co-Design Toolkit maintainer, you will:
- **Ensure Quality**: Review and approve toolkit content for accuracy and usability
- **Support Community**: Respond to user questions and facilitate discussions
- **Guide Development**: Help prioritize new tools and improvements
- **Maintain Standards**: Uphold evidence-based, user-centered design principles

### **Success Criteria**
By completion, you should be able to:
- ‚úÖ Navigate and understand the entire toolkit structure
- ‚úÖ Review and approve new tool content independently
- ‚úÖ Manage community interactions professionally and helpfully
- ‚úÖ Follow all maintenance procedures and quality standards

### **Knowledge Verification**

**Self-Assessment Quiz:**
```markdown
Answer these questions to verify your understanding:

1. What are the 4 main tool categories and their purposes?
2. What makes a tool "evidence-based" according to our standards?
3. What are the 5 phases of co-design and their typical duration?
4. How do we determine tool difficulty levels (Beginner/Intermediate/Advanced)?
5. What information must be included in source attribution?
6. What's our target response time for community questions?

Answers should align with documentation. Discuss any uncertainties with supervisor.
```

**Practical Exercise:**
- [ ] **Tool Navigation Test**: Find 3 tools for a hypothetical user scenario
- [ ] **Quality Assessment**: Review one existing tool and identify what makes it high-quality
- [ ] **Community Simulation**: Draft responses to 2-3 recent community questions (don't post yet)

## üë• Phase 3: Community Integration (Week 2-3)

### **Shadow Period Activities**

<details>
<summary><strong>üé≠ Learning Through Observation</strong></summary>

**Week 2 - Observer Mode:**
- [ ] **Issue Triage**: Shadow experienced maintainer during weekly issue review
- [ ] **Community Response**: Observe how experienced maintainers respond to questions
- [ ] **Content Review**: Watch experienced maintainer review new/updated tool content
- [ ] **Decision Making**: Observe how toolkit decisions are made and documented

**Week 3 - Guided Participation:**
- [ ] **Co-Review Tools**: Partner with experienced maintainer to review content together
- [ ] **Draft Responses**: Write responses with supervisor review before posting
- [ ] **Quality Checks**: Perform quality assurance tasks with guidance and feedback
- [ ] **Community Engagement**: Begin participating in discussions with backup support

</details>

### **Community Interaction Skills**

**Response Quality Standards:**
```markdown
Every community interaction should be:
‚úÖ Timely: Within 48 hours for initial response
‚úÖ Helpful: Provides actionable guidance or clear next steps
‚úÖ Professional: Respectful tone, proper grammar, clear communication
‚úÖ Evidence-Based: References toolkit content or authoritative sources when relevant
‚úÖ Encouraging: Supports user success and continued engagement

Practice Template:
"Hi [name], thank you for [specific contribution/question]. 
[Specific response to their need]
[Link to relevant resources if applicable]
[Clear next steps or invitation for follow-up]
Let me know if you have any other questions!"
```

**Common Scenarios Practice:**
- [ ] **New User Question**: "I'm new to co-design, where should I start?"
- [ ] **Tool Request**: "Can you create a tool for [specific need]?"
- [ ] **Bug Report**: "This link is broken / I found an error"
- [ ] **Unclear Documentation**: "I don't understand how to use this tool"
- [ ] **Methodology Question**: "Why do you recommend this approach over [alternative]?"

### **First Community Contributions**

**Supervised Activities (Week 3):**
- [ ] **Issue Response**: Respond to 2-3 straightforward questions with supervisor review
- [ ] **Link Maintenance**: Fix 3-5 broken links and update documentation
- [ ] **Minor Updates**: Make small content improvements based on user feedback
- [ ] **Quality Assurance**: Complete quality check on 1-2 tools independently

**Success Metrics:**
- Community responses are helpful and well-received
- Content changes follow quality standards and style guide
- No significant errors in technical edits
- Comfortable using GitHub workflow for changes

### **Content Review Certification**

<details>
<summary><strong>üìù Content Review Mastery</strong></summary>

**Review Competency Assessment:**

**Technical Review Skills:**
- [ ] **Source Verification**: Can validate all source citations and attributions
- [ ] **Methodology Assessment**: Can evaluate if tool follows evidence-based co-design principles  
- [ ] **Cross-Reference Check**: Can verify all internal links and integration sections
- [ ] **Format Compliance**: Can ensure tool follows standard template and style guide

**User Experience Review Skills:**
- [ ] **Clarity Assessment**: Can identify confusing or unclear instructions
- [ ] **Actionability Test**: Can determine if users can successfully implement the tool
- [ ] **Accessibility Check**: Can verify screen reader compatibility and visual accessibility
- [ ] **Completeness Verification**: Can confirm all necessary information is included

**Approval Authority Readiness:**
- [ ] **Quality Standards**: Consistently identifies content that meets/doesn't meet standards
- [ ] **Improvement Recommendations**: Provides specific, actionable feedback for content improvement
- [ ] **Risk Assessment**: Can identify content that might confuse users or provide poor guidance
- [ ] **Integration Awareness**: Understands how tools fit together and complement each other

</details>

### **Community Leadership Development**

**Independent Community Management:**
- [ ] **Issue Triage**: Independently categorize and label new issues
- [ ] **Response Ownership**: Handle routine questions without supervisor review
- [ ] **Escalation Judgment**: Know when to escalate complex issues to senior maintainers
- [ ] **Community Facilitation**: Help users connect with each other and find resources

**Proactive Community Support:**
- [ ] **Content Gaps**: Identify opportunities for new tools based on community feedback
- [ ] **User Success**: Follow up with users to ensure their questions were fully resolved
- [ ] **Process Improvement**: Suggest improvements to maintenance processes based on experience
- [ ] **Knowledge Sharing**: Help onboard newer community members and contributors

### **First Month Deliverables**

**Required Completions:**
- [ ] **Tool Review**: Complete independent review and approval of 1 new or updated tool
- [ ] **Community Engagement**: Successfully handle 5+ community questions/issues
- [ ] **Process Improvement**: Identify and document 1 potential improvement to maintainer processes
- [ ] **Knowledge Demonstration**: Successfully explain toolkit value and usage to a new user
