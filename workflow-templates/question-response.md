# Issue Triage Workflow
Quick daily process for handling new GitHub issues.

## Morning Triage (2-3 min per issue)
1. **Categorize**
   - [ ] Question → Add "question" label
   - [ ] Bug → Add "bug" label
   - [ ] Feature request → Add "enhancement" label
   - [ ] Documentation → Add "documentation" label

2. **Add Context Labels**
   - [ ] Category (users-stakeholders, technical-codev, etc.)
   - [ ] Difficulty (if relevant)
   - [ ] Phase (if relevant)

3. **Route**
   - [ ] Can answer now → Answer immediately
   - [ ] Need expertise → Tag relevant maintainer
   - [ ] Need info → Ask clarifying questions
   - [ ] Community input needed → Add "community discussion" label

## Standard First Responses
**For Questions:**
```
Thanks for your question! [Answer if possible, or...]
I'll look into this and get back to you within 48 hours.
```

**For Bugs:**
```
Thanks for reporting this. Could you please provide:
- What you were trying to do
- What unexpected thing happened
- Any error messages you saw
```

**For Feature Requests:**
```
Thanks for the suggestion! We'll review this during our next planning session and update you on potential inclusion.
```

# Question Response Workflow
Quick guide for consistent, helpful community responses.

## Response Checklist (5-10 min)
1. **Quick Assessment**
   - [ ] Read question carefully
   - [ ] Check if similar questions exist
   - [ ] Identify relevant tools/documentation

2. **Write Response**
   - [ ] Thank them for asking
   - [ ] Answer main question clearly
   - [ ] Link to relevant tools/docs
   - [ ] Invite follow-up if needed

3. **Final Check**
   - [ ] Answer is complete
   - [ ] Links are correct
   - [ ] Tone is helpful
   - [ ] Invites further engagement

## Response Templates
**Tool Usage Question:**
```
Thanks for asking about [tool]! 

[Direct answer to their question]

You might also find these helpful:
- [Related tool link]
- [Relevant documentation]

Let me know if you need any clarification!
```

**General Question:**
```
Thanks for your question! 

[Clear, concise answer]

For more details, check out:
[Relevant links]

Feel free to ask if you need anything else!
